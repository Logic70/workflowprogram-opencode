ClaudeCode 到 OpenCode 适配核心关注点

本文总结 WorkflowProgram 从 ClaudeCode 元工作流迁移到 OpenCode 独立版本时的关键设计判断。 核心原则是：不做 ClaudeCode 目录结构兼容层，而是用 OpenCode 原生的 commands、agents、plugins、runtime 和项目本地安装模型重新落地同等产品能力。

OpenCode only Project-local materialization Global bootstrap only Layered contracts

一、核心关注点

加载边界

必须区分 WorkflowProgram 自身如何被 OpenCode 加载，以及 WorkflowProgram 生成的目标工作流如何被 OpenCode 加载。前者是产品包契约，后者是目标 bundle 契约。

命名空间

/wp-* 归 WorkflowProgram 产品命令所有；目标工作流命令不能占用该命名空间，package plugin 和 target plugin 也不能复用逻辑标识。

安装体验

完整全局安装易用但污染风险高；纯项目安装隔离好但新项目体验差。最终采用全局轻量 bootstrap、用户级 cache、项目本地物化三层模型。

验证真实性

runtime smoke 只能证明 Python 主链可运行；host smoke 才能证明 OpenCode 可发现命令和插件；target host smoke 才能证明生成物对宿主可见。

二、主要难点与解决方案

难点 风险 解决方案
Claude skill、command、hook 与 OpenCode agent、command、plugin 不是一一映射 把 ClaudeCode 的加载模型错误复制到 OpenCode，导致 schema 和交付形态混乱 只迁移产品能力，不迁移 Claude 目录结构；OpenCode 版以 .opencode/commands、.opencode/agents、.opencode/plugins 为宿主真源。
WorkflowProgram 产品包和目标工作流都可能生成 OpenCode 资产 产品包插件、目标插件、产品命令、目标命令互相覆盖 定义三根路径：WP_PACKAGE_ROOT、TARGET_ROOT、RUN_ROOT；目标命令禁止 wp-*；target plugin 禁止使用 package plugin 标识。
OpenCode 可能读取全局配置、Claude 目录或 oh-my-opencode 资产 用户以为使用的是当前项目 WorkflowProgram，实际命令或 skill 来自其它路径 /wp-doctor 增加 host source inventory、shadowing 检测、WSL/Windows 路径诊断和 reload guidance。
项目本地安装每个项目都要执行一次 用户体验不友好，安装命令难记 新增全局轻量 bootstrap：全局只安装 /wp-install、/wp-status、/wp-upgrade、/wp-uninstall，完整 package 仍物化到项目本地。
Python 依赖可能缺失，尤其是 PyYAML 安装器或 runtime 无法启动 声明 requirements.txt 和 requirements.lock.txt；项目安装可创建 .workflowprogram/package/.venv；bootstrap 可先预置 venv 再运行 package deploy。
真实 OpenCode 执行依赖 provider/API，可用性不稳定 测试假阳性或把环境问题误判为设计失败 测试结果分为 PASS、FAIL、ENVIRONMENT-SKIP；无 API 时不声称真实宿主执行成功。

三、最终架构取舍

最终选择不是“完整全局安装”，也不是“每个项目都手动执行长命令”，而是： 全局轻量 bootstrap + 用户级版本化 package cache + 项目本地 materialized install。

方案 结论 原因
完整全局安装 不采用 易造成命令污染、版本串扰、项目间不可复现和 Claude/OpenCode 资产混读。
纯项目本地安装 保留为基础能力 隔离性最好，但新项目首次使用不够友好。
全局轻量 bootstrap 采用 用户只需全局安装一次部署器；每个项目仍获得完整、可诊断、可回滚的本地安装。

四、验收口径

  • 全局 bootstrap 只出现部署命令，不出现完整 /wp-develop 产品生命周期命令。
  • 新项目执行 /wp-install 后，项目内出现完整 .opencode/* 和 .workflowprogram/package/*。
  • 项目安装后重启 OpenCode，完整 /wp-* 命令和 package agents 可见。
  • runtime smoke、host integration smoke、target host smoke 和 bootstrap smoke 各自独立报告，不互相替代。
  • 所有写入型 intent 都走 candidate、managed apply、lock 和 rollback manifest。
